## Core Theory

### Isaac Sim Architecture

```
Application Layer: Python API, ROS 2 Bridge, Extensions
├─ Simulation Core: PhysX 5 (GPU), Replicator (synthetic data)
├─ Rendering: RTX Ray Tracing, Path Tracing
├─ Platform: Omniverse Nucleus (collaboration, USD versioning)
└─ Hardware: CUDA, Tensor Cores, RT Cores
```

**Key Components**:
- **USD (Universal Scene Description)**: Pixar's open format for 3D scenes
- **Omniverse Nucleus**: Cloud/local server for asset sharing
- **PhysX 5**: NVIDIA's GPU-accelerated physics engine
- **Replicator**: Synthetic data generation with randomization
- **Extensions**: Plugin system for custom tools

### USD Workflow

USD organizes scenes hierarchically:

```python
from pxr import Usd, UsdGeom

# Create stage (root scene)
stage = Usd.Stage.CreateNew("my_scene.usd")

# Add robot as reference
robot_prim = stage.DefinePrim("/World/Robot", "Xform")
robot_prim.GetReferences().AddReference("robot.usd")

# Add ground plane
UsdGeom.Mesh.Define(stage, "/World/Ground")

# Save
stage.Save()
```

**USD Advantages**:
- Lazy loading (only load visible assets)
- Non-destructive edits (layer composition)
- Collaborative workflows (multiple users, branching)

### PhysX 5 vs Traditional Engines

| Feature | PhysX 5 (Isaac Sim) | ODE (Gazebo) | Bullet |
|---------|---------------------|--------------|--------|
| **GPU Acceleration** | Yes (thousands of actors) | No | Limited |
| **Soft Bodies** | Yes (FEM solver) | No | Basic |
| **Determinism** | Optional (with settings) | Yes | Varies |
| **Contact Solver** | TGS (temporal) | PGS | Sequential Impulse |
| **Parallelization** | Massive (4096+ envs) | None | Multi-thread |

## Hands-on Code

**File**: `examples/module-3-isaac/chapter-1/create_scene.py`

```python
from isaacsim import SimulationApp
simulation_app = SimulationApp({"headless": False})

from omni.isaac.core import World
from omni.isaac.core.objects import DynamicCuboid
from omni.isaac.core.utils.nucleus import get_assets_root_path

# Create world
world = World()

# Add ground plane
world.scene.add_default_ground_plane()

# Add robot from Nucleus
assets_root = get_assets_root_path()
robot_usd = assets_root + "/Isaac/Robots/Franka/franka.usd"
world.scene.add(robot_usd)

# Run simulation
world.reset()
while simulation_app.is_running():
    world.step(render=True)

simulation_app.close()
```

## Common Pitfalls

### Pitfall 1: Out of GPU Memory
**Symptom**: Crash when loading large scenes or many parallel environments.
**Fix**: Reduce visual fidelity, use level-of-detail (LOD), batch smaller.

### Pitfall 2: Non-Deterministic Physics
**Symptom**: Same initial conditions give different results.
**Fix**: Enable deterministic mode in PhysX settings (trades speed for reproducibility).

### Pitfall 3: Slow USD Loading
**Symptom**: Scene takes minutes to load.
**Fix**: Use instancers for repeated objects, flatten layer stack, cache Nucleus assets.
